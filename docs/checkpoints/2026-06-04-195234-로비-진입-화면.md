## 하던 일
사용자 "테스트시 처음 방을 만드는거부터 해야지" — URL ?room= 직접 입력 대신 로비(방 만들기/코드 입장)부터
시작하는 흐름 추가. 디자인 원본 lobby.jsx를 백엔드(방 목록 API 없음, 방 코드 기반)에 맞게 포팅.

## 내린 결정
- 화면 전환을 App.jsx에서 관리: session 없으면 로비, 있으면 RoomView. URL ?room=&me= 는 공유/초대 호환.
- 방 목록은 백엔드에 없어 localStorage(rooms.js)에 보관 — 재입장 바로가기.
- 방 코드는 클라이언트 생성(makeRoomCode). 방은 백엔드 /ws/{code} 첫 연결 시 자동 생성.
- 방장 등록: "새 방 만들기"로 들어간 host만 입장 시 set_meta(owner·destination·dates)+set_accommodation(거점).
  참여자(코드 입장)는 메타 미전송. 방장 표시·확정 버튼은 백엔드 state.owner==me로 판정.
- 프론트 단위테스트 인프라 없음 → 빌드+SSR 렌더+백엔드 WS 프로브로 검증.

## 검증
- vite build 그린, SSR 로비 렌더 OK(1858자, 마커 갱신), dev 신규 모듈 변환 200, CI 통과.
- WS 프로브: 방 생성 액션 백엔드 반영(owner=민수·destination=제주·dates·숙소=애월 거점) + 방장 확정=True.
- 앞서 실서버 구독 봇 카드 emit OK(네거리식당 등).
