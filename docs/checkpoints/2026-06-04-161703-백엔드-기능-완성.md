**71개 테스트 통과**(SDK·키 없이 실행, 실서버 WS 통합 포함).

## 내린 결정
- LLM 실행: **Agent SDK → Messages API 직접**(`ApiAgentRunner`) + **cli/api 실행 모드 토글**. SDK는 로컬/CLI(`cli_runner`)만 — 서버리스 배포 자유.
- 영속: **앱 상태(Supabase)가 진실의 원천**, SDK 세션은 휘발성 캐시. 재개=스냅샷 주입.
- 동선: **결정적 NN+2-opt**(LLM이 순서 안 짬). 트리거: **명시적 호출만**. 의사결정: **방장 단독 확정**.
- 출력 계약: **present_* 카드**(ToolSpec, emit→방 브로드캐스트). 일정 카드는 `working_itinerary`로 영속.
- 외부 의존(Anthropic·Kakao·Supabase·SDK)은 전부 인터페이스 seam으로 격리 → 키 없이 테스트.
- 데이터=Kakao(동선·검색) + Google(영업시간·리뷰, 추후). 용어 기준 `docs/TERMS.md`, 도메인 `CONTEXT.md`.

3. **실키 런타임 검증** — Anthropic Messages API, Kakao REST, Supabase RowStore 어댑터(현재 seam).
4. 에러/로딩 풀세트, 확정본↔작업본 뷰 분리, 권한 엣지케이스.

## 메모
- 액션 프로토콜: `{"action":"add_candidate|remove_candidate|set_meta|set_accommodation|set_preference|confirm_itinerary",...}` (채팅은 `{"speaker","text"}`). 응답: `{"type":"state",...}` / 에러 `{"type":"error",...}`. 카드: `{"type":"card","card":{...}}`.
- seam 목록: `api_runner.anthropic_model_client`, `kakao.KakaoClient.from_env`/`make_httpx_get`, `state.SupabaseStateStore`(RowStore 주입), `cli_runner.CliAgentRunner`(SDK).
- 작업 러너 툴: search_places·travel_time(Kakao 키 있을 때)·order_route(결정적·항상)·present_place_options/itinerary/map.
- 환경: Python venv `.venv`(pytest 9, pytest-asyncio, fastapi, uvicorn, websockets, httpx). node v24/npm 11 가용(Vite 가능). `anthropic`·`supabase`·`claude_agent_sdk`·Kakao 키는 미설치/미설정(런타임 seam).
- 디자인 web/는 데모 데이터(멤버·코스 용어 → 실연결 시 여행자·일정으로 정합). 원본 핸드오프 `design/web/`(HANDOFF.md·UX_REVIEW.md 포함) 보존.
