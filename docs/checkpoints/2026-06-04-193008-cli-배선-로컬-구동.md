## 하던 일
사용자 "일단 로컬에 띄워서 확인하고 배포하자" → 봇 구동 방식은 **구독(CLI) 경로** 선택.
main.py가 BACKEND=cli로 키 없이 봇을 구동하도록 배선하고, 백엔드+프론트를 로컬에 띄움.

## 내린 결정
- routing.HeuristicClassifier 신설: LLM 호출 없이 키워드(/일정·검색·동선·짜줘 등)로 단순/작업 분기.
  CLI 경로에서 분류만을 위한 LLM 호출을 피한다(ApiClassifier는 API 키 필요).
- factory.build_default_runner(backend="cli"): 휴리스틱 분류기 + CliAgentRunner(단순/작업).
  api 경로는 기존 유지. 둘 다 SDK/키 없이 합성됨(지연 import).
- main.py: BACKEND 환경변수로 경로 선택(기본 api, 로컬 cli).
- TDD로 진행(휴리스틱 3 + factory cli 1 테스트). 76 passed, 1 skipped(cli e2e).
