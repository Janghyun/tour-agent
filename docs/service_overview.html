방 (Room)
여행자들과 봇이 모인 공간. 자체 웹앱.
하나의 여행 = 하나의 방
여행자 · 방장
사람 멤버. 방장은 일정을 확정할 수 있는 유일한 역할.
방장만 확정 권한
후보 장소 풀
검색 결과에서 "추가"한 방문 후보. 방 전체 공유.
일정의 입력 재료
일정 · 확정 일정
방마다 단일 작업 일정. 방장이 확정하면 그 시점 스냅샷이 확정 일정.
작업본 1개 + 확정 스냅샷
동선 · 숙소
박별 숙소 기준 이동. 동선은 결정적 알고리즘이 계산.
숙소 = 박별 거점
선호
개인 취향(좋아요/싫어요). 방에 공유돼 추천에 반영.

핵심 흐름

사용자가 실제로 겪는 것
A장소 검색 → 후보 추가
  1. @봇으로 검색
  2. 봇이 옵션 카드 제시
  3. "추가" 누른 것만 풀에 등록
B일정 설계
  1. @봇 "일정 짜줘"
  2. 작업 중 표시 + 큐잉(논블로킹)
  3. 설계·동선·검증 → 일정 카드
C수정 · 되물음
  1. "우도를 둘째 날로"
  2. 작업 일정 수정
  3. 모호하면 한 번 확인
E단순 질의 즉답
  1. "내일 비 와?"
  2. 방 맥락(제주·날짜) 반영
  3. 바로 답

왜 이렇게
