3 우리 코드는 어떻게 갈리나

같은 AgentRunner 인터페이스 뒤에서, Backend 옵션이 구현을 고릅니다. 봇 행동을 정하는 ToolSpec·시스템 프롬프트는 양쪽이 공유합니다.

interface AgentRunner · ToolSpec · system prompt  —  양쪽이 공유

4 핵심은 두 가지

① 서브프로세스
공식 Agent SDK는 claude-code 바이너리를 서브프로세스로 띄웁니다. Workers는 샌드박스라 프로세스를 못 띄웁니다. 그래서 Workers에선 SDK를 못 쓰고 API를 직접 부릅니다. (Python Workers도 마찬가지 — 언어 문제가 아니라 런타임 문제)
② 상태(state)
로컬은 한 프로세스 메모리에 방별 세션과 락을 들고 있습니다. Workers의 isolate는 요청마다 생겼다 사라져 메모리가 안 남습니다. 그래서 상태는 Supabase로 빼거나, 방별 코디네이션은 Durable Object(객체=방, 단일 스레드라 락이 공짜)로 옮깁니다.
이 문서는 우리 백엔드 기준의 비교입니다. 두 모드 모두 같은 AgentRunner/ToolSpec을 공유하므로, 배포 타깃은 나중에 바꿔도 코드 대부분은 그대로입니다. · 토글로 모드를 전환해 보세요.
