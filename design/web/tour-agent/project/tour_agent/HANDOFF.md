## 구현팀 인계 (디자인 범위 밖 — 실제 빌드 시 처리)

1. **프론트 인텔리전스 제거**: `data.jsx`의 `classifyIntent`(정규식)·`Scenario.*`(하드코딩 페이로드)는 데모용. 실제는 게이트 B(Haiku)+서브에이전트가 백엔드에서 판단하고 프론트는 `present_*` 카드 페이로드만 수신·렌더. 의도분류 코드는 폐기 대상.
2. **지도 실연동**: 현재는 손그림 제주 일러스트(고정 viewBox 0~600 좌표). Kakao Map JS + 위경도로 교체 시 핀 좌표계·동선 폴리라인·랜드마스 전부 재구현. 다지역(부산·통영) 방은 제주 일러스트가 그대로 떠 깨지므로 지도 컴포넌트 재작성 전제.
3. **실데이터 필드 갭**: Kakao Local 영업시간 미반환 → WebSearch 교차확인 + "확인 필요" 톤(슬롯 마련됨). 거리("성산 0.8km")는 숙소 기준 동적 계산값으로 교체(현재 정적). 평점·설명은 출처별 제공 여부 달라 폴백 표기 필요.
4. **에러·로딩 풀세트**: 일정 생성 장시간(1~2분) 실패/타임아웃/부분결과/재시도, WebSocket 재연결, 큐 작업 실패, 권한 만료 등은 추가 설계 필요(이번엔 링크·검색·후보 가드만).
5. **확정본/작업본 뷰 분리**: 확정 후에도 작업본은 계속 변함 → "확정본 보기 ↔ 작업본 보기" 전환/디프 뷰 설계 필요(현재 작업본 단일 표시 + 확정 스냅샷 안내 문구).
6. **권한 엣지케이스**: 방장 부재 시 확정 데드락, 동시 슬롯 수정 충돌 표시, 비방장의 "확정 요청" 신호 경로.
7. **데모 제거 대상**: Tweaks 패널, 역할(여행자/방장) 토글, 데모 시간 로직(`clock` 14:0x), 한글 쿼리 외부 링크.
8. **외부 링크 후보 추가 신뢰성**: 임의 블로그/인스타에서 좌표·카테고리 추출 난도 높음. place_id 기반 딥링크 가능 여부(Kakao 가능, 네이버·구글 제약) 확인.
